# v1.5 live test — extension verified against a real Sophia Stack

> File: docs/v1.5-live-test.md · Date: 2026-06-27
> Ran the extension end-to-end inside an actual Sophia Stack v1.5 deployment
> (local), covering the full owner-safe lifecycle. Evidence for the WS7 demo.

## Setup
- Stack server booted locally (`scripts/serve.mjs`) with the default sample site.
- Extension loaded from `.sophia-data/extensions/sophia-seo-suite` (the public-repo
  copy; `requires` relaxed to `>=1.0.0` **only** for this run because the running
  Stack still self-reports `1.0.0` — see "Finding" below; the committed manifest
  stays `>=1.5.0`).

## Finding for the Stack (joint-cut interlock)
The install endpoint **correctly enforces** `requires.sophiaStack` — it refused the
`>=1.5.0` extension with *"needs Sophia Stack >=1.5.0 (you have 1.0.0)"*. All v1.5
**features** are present (panels, `ctx.versions`, native SEO render), but the
running Stack's `VERSION` (`src/version.mjs`, surfaced from `package.json 1.0.0`)
hasn't been bumped. **Action (Stack):** bump `VERSION` to `1.5.0` for the joint cut
so v1.5-gated extensions (including one-click "Add Sophia SEO Suite") install. This
is the only thing standing between "all features shipped" and "one-click install
works unmodified." Raised in the coordination inbox.
